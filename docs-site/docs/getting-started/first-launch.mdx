# First Launch Setup

On first launch, Row-Bot asks for one working model path. Everything else can
be finished later from Setup Center or Settings.

<Screenshot
  id="first-launch-setup-wizard"

  caption="The setup wizard keeps local, hosted, subscription, and custom endpoint paths separate."
/>

## Choose A Model Path

- Local/Ollama keeps inference on your machine when the selected model and tools
  are local. Install Ollama, start it, and choose an exposed local model.
- Provider/API-key paths connect hosted providers such as OpenAI, Anthropic,
  Google AI, xAI, MiniMax, OpenRouter, Atlas Cloud, and Ollama Cloud.
- ChatGPT / Codex and Claude Subscription use local sign-in or external CLI
  flows where supported instead of a traditional provider API key.
- Custom/self-hosted endpoints support OpenAI-compatible servers such as LM
  Studio, vLLM, llama.cpp, LocalAI, LiteLLM, SGLang, oMLX, or private gateways.

Row-Bot stores provider identity with the model choice, so two models with the
same display name are not treated as interchangeable.

## Continue Setup

After model setup, Setup Center can guide you through documents, workflows,
Designer Studio, Developer Studio, channels, accounts, tools, plugins, MCP, and
voice.

<Screenshot
  id="setup-center"
  alt="Row-Bot Setup Center showing resumable setup checklist cards."
  caption="Setup Center is optional after the first working model is configured."
/>

You can skip any setup area and return later. For privacy-sensitive areas such
as channels, provider credentials, MCP servers, plugins, and Developer Studio,
review the relevant settings before enabling background behavior.

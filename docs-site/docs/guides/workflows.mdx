# Workflows

Workflows let Row-Bot run repeatable or scheduled tasks in the background. They
can use prompts, steps, conditions, approvals, model overrides, tool overrides,
skills, channels, webhooks, completion triggers, and safety modes.

The docs automation pipeline will generate screenshots for:

- the Workflows tab;
- workflow cards;
- workflow editor states;
- run status and history;
- approval handling.

## First Workflow

Start with a simple prompt-style workflow, keep it disabled while editing, then
run it manually once before enabling a schedule.

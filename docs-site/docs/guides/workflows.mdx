# Workflows

Workflows let Row-Bot run repeatable or scheduled tasks in the background. A
workflow can include prompts, steps, conditions, approvals, model overrides,
tool overrides, skills, delivery channels, webhooks, completion triggers, and
safety modes.

## Create A First Workflow

1. Open Home, then Workflows.
2. Create a workflow with a clear name and a single prompt step.
3. Keep it disabled while editing.
4. Choose the model, tools, channels, and approval behavior.
5. Run it manually once before enabling a schedule.

Use approvals for workflows that write files, send messages, call external
tools, or change workspace state. Keep scheduled workflows narrow and observable
until you trust their output.

## Delivery And Run History

Workflows can deliver results in the app or through configured channels. If a
run waits for approval, resolve it in the app before expecting downstream
delivery. Review run history after changing model or tool settings because
provider behavior can change the workflow output.

Architecture

How Row-Bot stays local-first.

These diagrams map the main systems behind Row-Bot: the checkpointed agent orchestrator, context assembly, memory graph, workflows, channels, native Computer Use, Developer and Designer Studios, safety boundaries, and self-management loops. Durable state stays local; external services and optional runtimes remain explicit and opt-in.

Explore diagrams Read technical reference
Local state

Conversations, memory, media, tasks, wiki vaults, and project assets live on the user's machine.

Opt-in models

Local runtimes and self-hosted endpoints stay distinct from hosted provider APIs, Ollama Cloud, and ChatGPT / Codex.

Trust boundaries

Tool results, web pages, channel attachments, and remote MCP servers are treated as untrusted context.

User control

Destructive tools, settings changes, workflow actions, native Computer Use, and external delivery paths pass through guardrails.

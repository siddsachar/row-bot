Privacy by architecture

Local data first.
Providers by choice.

Row-Bot has no account system or app telemetry pipeline. Local runtimes can work offline, and hosted providers, cloud embeddings, messaging channels, tunnels, and online tools remain opt-in. Secrets use the operating-system credential store when available instead of local metadata files.

Before you install

Practical answers.

Row-Bot is broad, but the operating model is simple: local application, explicit providers, durable local records, and reviewable actions.

Is Row-Bot free?

Yes. Row-Bot is Apache 2.0 licensed and does not require a Row-Bot subscription. Local models can be free to run on your own hardware; hosted providers, external services, tunnels, and some tools may have their own charges.

Can it run entirely locally?

Yes, when you select a local runtime and local tools. Web search, hosted models, cloud embeddings, online channels, and remote access naturally require network connections and are opt-in.

What is Mobile Companion?

It is a phone-native web interface served by your running Row-Bot desktop host. Pair a phone with a short-lived QR link over an approved route, then use chat, activity, workflows, approvals, knowledge, and selected settings.

Which model providers are supported?

Row-Bot supports Ollama, custom OpenAI-compatible runtimes, major API providers, routing services, xAI Grok OAuth, ChatGPT/Codex, Claude Subscription, Ollama Cloud, OpenCode providers, and additional provider-qualified integrations. See the detailed features page for the current inventory.

What reaches a hosted model?

The selected provider receives the current model-visible conversation and tool context needed for that request. Row-Bot’s durable conversation database, memory graph, profiles, goals, and project records remain on your machine unless you deliberately use a tool or provider that sends specific data elsewhere.

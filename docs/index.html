Local-first architecture

Your machine is the system of record.

Conversations, profiles, goals, knowledge, project context, workflow history, and paired-device records live locally. Hosted providers and networked tools receive the context required only for requests you explicitly choose.

Before you install

Practical answers.

A local application, explicit providers, durable local records, and reviewable actions.

Is Row-Bot free?

Yes. Row-Bot is Apache 2.0 licensed and does not require a Row-Bot subscription. Local models can be free to run on your hardware; hosted providers and external services may have their own charges.

Can it run entirely locally?

Yes, when you select a local runtime and local tools. Hosted models, cloud embeddings, web tools, online channels, and remote access require network connections and remain opt-in.

Which desktop platforms and model providers are supported?

Row-Bot supports Windows 10/11, macOS, and Linux. Model paths include Ollama, major hosted APIs, subscription providers, routing services, and private OpenAI-compatible endpoints; see the maintained provider inventory for details.

What reaches a hosted provider?

The selected provider receives the current model-visible conversation and tool context needed for that request. Durable conversations, knowledge, profiles, goals, and project records stay on your machine unless you deliberately send specific data through a provider or networked tool.

What is Mobile Companion?

Mobile Companion is a phone-native web interface served by Row-Bot running on your computer. It connects through a route you choose and is not a standalone iOS or Android download; the host computer must remain running and reachable.

Download and install

Your AI workspace. On your machine.

Install Row-Bot, choose a local or hosted model path, and continue through the resumable Setup Center. Mobile Companion connects back to this running computer; it is not a separate cloud service.
